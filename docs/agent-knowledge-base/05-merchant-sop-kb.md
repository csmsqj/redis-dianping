# 商家运营 SOP 知识库

本文件面向商家和平台运营，用于减少人工解释和重复培训。Agent 可以根据本文件回答“商家如何提高转化”“怎么写优惠券规则”“门店信息怎么维护”等问题。

## 商家页面转化的核心

用户打开门店页面后，通常只看几件事：

- 店名和类目是否匹配。
- 图片是否真实清晰。
- 评分是否稳定。
- 评论数量是否足够。
- 人均是否符合预算。
- 距离和地址是否方便。
- 有无优惠券。
- 营业时间是否合适。

商家运营的目标不是把页面写得很长，而是减少用户决策的不确定性。

## 门店基础信息维护

### 必填项

- 店名。
- 类目。
- 地址。
- 商圈。
- 营业时间。
- 人均价格。
- 联系方式。
- 门店图片。
- 服务项目或菜品特色。

### 维护原则

- 地址要能被用户直接导航。
- 营业时间要及时更新，尤其是节假日。
- 图片要真实，不要全部使用过度修图。
- 人均价格要接近真实消费。
- 项目说明要避免隐藏费用。

### Agent 回复模板

> 门店信息维护的重点是降低用户到店前的不确定性。优先更新地址、营业时间、人均、图片和优惠说明，这些信息会直接影响用户是否下单。

## 图片运营 SOP

### 美食类

建议上传：

- 招牌菜。
- 店内环境。
- 套餐内容。
- 菜单或价格参考。
- 门头图。

避免：

- 过度滤镜。
- 与实际菜品不一致。
- 只上传局部特写，用户看不出份量。

### 服务类

建议上传：

- 环境。
- 操作区。
- 项目说明。
- 作品图。
- 卫生细节。

避免：

- 只放宣传海报。
- 没有真实空间图。
- 作品图没有说明价格范围。

## 优惠券配置 SOP

### 活动标题

标题要让用户一眼知道利益点。

好例子：

- 50 元代金券。
- 双人烤肉套餐。
- 工作日 KTV 3 小时欢唱。
- 美甲纯色单次体验。

不建议：

- 超值优惠。
- 精品套餐。
- 到店惊喜。

### 规则说明

规则要包含：

- 有效期。
- 可用时段。
- 适用门店。
- 是否需要预约。
- 是否可叠加。
- 是否限制节假日。
- 是否含服务费。
- 退款规则。

### 秒杀券配置

秒杀券适合引流，但要控制风险：

- 库存不要超出实际承接能力。
- 开始时间要明确。
- 限购规则要明确。
- 到店核销流程要提前培训。
- 活动结束后及时复盘。

## 评价运营 SOP

### 好评处理

- 感谢用户。
- 提取用户提到的亮点。
- 邀请下次体验。

模板：

> 感谢你的认可，很开心你喜欢我们的服务和环境。我们会继续保持，也欢迎下次再来体验新的活动。

### 中差评处理

- 先表达重视。
- 不与用户争辩。
- 说明将核实。
- 给出处理路径。

模板：

> 很抱歉这次体验没有达到你的预期。我们会根据你反馈的时间、项目和服务环节进行核实，也欢迎通过平台客服补充信息，我们会尽快处理。

### 不建议的回复

- “你说的不属实”。
- “我们店一直这样”。
- “你自己没看规则”。
- “不满意别来”。

## 客服省力策略

把重复问题提前写进页面和知识库：

- 停车怎么停。
- 是否需要预约。
- 能不能开发票。
- 儿童能不能使用。
- 宠物能不能进入。
- 是否支持包间。
- 周末是否可用。
- 是否有最低消费。
- 是否有服务费。

这些问题越早被回答，客服压力越小。

## 商家活动复盘

每次活动结束后，商家应复盘：

- 曝光量。
- 点击量。
- 领券量。
- 核销量。
- 到店高峰。
- 用户投诉点。
- 退款原因。
- 差评关键词。

Agent 可以根据复盘数据生成下一次活动建议，但如果没有数据，只能给通用建议。

## 面向平台的价值

把商家运营 SOP 放入知识库后，平台可以用 Agent 自动回答新商家培训问题，减少运营人员反复解释。也可以把标准回复给客服使用，提升回复一致性。
