# 黑马点评 Agent 知识库说明

这套资料用于构建“黑马点评 AI 导购助手”的第三个能力：基于知识库回答用户、商家、运营人员的高频问题。它不依赖实时数据库查询，也不直接调用后端 API；实时数据由“附近店铺查询”和“优惠券助手”两个工具能力处理。

## 适用目标

- 减少用户反复咨询：怎么找店、怎么筛选、怎么用券、怎么判断是否适合自己。
- 减少商家运营成本：门店资料怎么维护、活动怎么配置、评价怎么处理、客服怎么统一口径。
- 减少开发耦合：知识问答放在 Dify 或 RAGFlow 中维护，不频繁改 Java 后端或 Nginx 前端。

## 推荐导入方式

### Dify

把本目录中的 Markdown 文件导入 Dify 知识库。建议按文件切分，并给文档打标签：

- `user-guide`：用户使用说明、筛选规则、消费建议。
- `coupon-guide`：优惠券通用规则、核销前提醒。
- `merchant-sop`：商家运营 SOP、客服口径。
- `agent-prompt`：智能体系统提示词、回答边界。

然后在 Chatflow 中配置：

1. 用户问题进入意图判断。
2. 如果是“附近有什么店”“查这家有没有券”，走后端 API 工具。
3. 如果是“怎么选店”“券怎么用”“商家怎么提高转化”“为什么距离排序不准”，走知识库检索。
4. 最终回答时优先给用户下一步按钮，如“查附近店铺”“查优惠券”“换一批”“查看使用规则”。

### RAGFlow

如果你后续使用 RAGFlow，建议把 Markdown 作为主要知识库文件导入；RAGFlow 更适合处理长文档、FAQ、SOP 和复杂规则。Dify 可以作为对话入口，通过 HTTP 请求或工作流节点调用 RAGFlow 检索结果。

## 数据边界

这套知识库不存储真实用户隐私，不包含真实订单，不复制真实平台评论。它提供的是可解释的业务规则、问答模板、运营指南和导购话术。

实时状态必须通过后端接口确认：

- 某店是否存在。
- 某店当前评分、人均、距离。
- 某店是否有券。
- 秒杀券是否还有库存。
- 用户是否登录、是否已下单。

知识库可以回答：

- 如何选择附近店铺。
- 不同场景适合看哪些指标。
- 优惠券常见使用规则。
- 商家如何配置活动。
- 用户投诉、退款、预约、排队等标准答复。

## 文件清单

- `00-agent-blueprint.md`：智能体总设计。
- `01-screen-and-user-flow.md`：结合截图的界面理解与交互设计。
- `02-shop-discovery-kb.md`：附近店铺与筛选知识。
- `03-coupon-kb.md`：优惠券助手知识。
- `04-category-playbooks.md`：美食、KTV、丽人等类目导购话术。
- `05-merchant-sop-kb.md`：企业省力的运营 SOP。
- `06-customer-faq-kb.md`：用户常见问题。
- `07-rag-qa-seed.md`：可直接导入的问答种子。
- `dify-agent-system-prompt.md`：Dify 智能体系统提示词。
- `hmdp-agent-tools.openapi.yaml`：前两个实时功能的 OpenAPI 工具草案。

## 建议的第三个功能名称

推荐命名为：`消费决策与商家运营知识问答`。

这个名称比“随便问问”更清晰：它面向用户能做消费决策，面向企业能减少客服与运营解释成本。
